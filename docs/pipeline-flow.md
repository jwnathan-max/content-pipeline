# 파이프라인 흐름 & UI 구조

> 필요 시에만 읽으세요.

## 파이프라인 흐름

```
[채널 관리 탭]
  → yt-dlp로 채널 영상 수집 (최근 168시간)
  → processed_videos DB 대조 → 중복 배지 표시

[콘텐츠 생성하기 버튼] 또는 [URL 직접 입력]
  ↓
Step 1: 자막 추출 (youtube-transcript-api)
  우선순위: 한국어 수동 → 한국어 자동 → 영어(번역)
  에러: 없음→수동입력창 / 비공개→에러알림 / 타임아웃→3회재시도
  중복: processed_videos에서 video_id 확인 → 재생성 or 기존 사용 선택
  ↓
Step 2: AI 콘텐츠 생성 (Claude claude-sonnet-4-5, tool_use 방식)
  단일 호출로 SMS + 블로그 + 인스타 3포맷 동시 생성 (JSON)
  자막 8000자 초과 시: 청크 요약 → 재통합 → 최종 생성
  ↓
Step 3: Streamlit 에디터
  문자 탭: EUC-KR 90바이트 이하=SMS / 초과=LMS 실시간 표시
  블로그 탭: 마크다운 좌우 분할 미리보기 + 블로그 이미지 생성(1200×630)
  인스타 탭: 캡션·해시태그 편집 (이미지 생성은 Phase 2)
  ↓
[배포]
  Ghost Admin API → 발행 URL + DB 기록
  Instagram Graph API → 게시물 링크 + DB 기록 (Phase 2)
  알리고 API → 문자 발송 / 미연동 시 TXT 파일 다운로드 (Phase 2)
```

## Streamlit 탭 구조 (app.py)

```
탭1: 채널 관리    — 채널 추가/삭제, 활성화 토글
탭2: 영상 수집
  서브탭1: 채널 구독 영상 — yt-dlp 기반, 최근 168시간
  서브탭2: 키워드 검색   — 프리셋 6개 + 직접입력 + 일괄검색
  공통: [콘텐츠 생성하기] 버튼
탭3: 콘텐츠 생성  — URL 입력 or 수집 탭 연동, Step 1~3 진행
  서브탭: 문자 | 블로그 | 인스타 에디터 + 배포 버튼
탭4: 처리 이력    — 최근 50개 목록 (Phase 3 대시보드 예정)
```
